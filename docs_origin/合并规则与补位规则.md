# 合并规则与补位规则

## 一、合并规则

### 1. 合并触发

- 玩家可上下左右滑动 1 个食材，与相邻食材交换位置。
- 交换后，若出现 3 个及以上同等级食材横向或纵向连成直线，则触发合并。
- 未形成有效三连时，本次交换撤销，2 个食材回到原位置。

### 2. 合并结果

- 每个有效合并组只生成 1 个更高一级的食材。
- 新食材生成在本次交换涉及的位置中，由实际触发合并的那个格子承接。
- 参与合并的其他食材全部消失，留下空格。

### 3. 连锁规则

- 一次滑动触发合并后，先完成本轮合并结算。
- 之后进入补位。
- 若补位完成后再次形成新的三连，则继续触发下一轮合并。
- 流程循环为：`合并 → 补位 → 再检测 → 再合并`，直到棋盘稳定。

***

## 二、口味标签规则

### 1. 基础定义

- 口味改为“标签”机制。
- `原味` 视为无标签，不参与口味竞争。
- 合并时只统计本次参与合并的非原味口味。

### 2. 单一口味继承

当本次合并只涉及 1 种非原味口味时，新食材直接继承该口味：

| 合并前          | 合并后 |
| ------------ | --- |
| 原味 + 草莓      | 草莓  |
| 原味 + 抹茶      | 抹茶  |
| 草莓 + 原味 + 原味 | 草莓  |

### 3. 多口味冲突结算

当本次合并同时涉及 2 种及以上非原味口味时：

1. 统计每种口味在本次合并中的出现数量
2. 按数量作为权重随机保留 1 种口味
3. 新食材只继承被抽中的那一种口味
4. 其他未被抽中的口味直接消失

### 4. 提示动画

- 当进入多口味冲突结算时，在合并落点位置展示一个小转盘动画
- 动画时长建议 `0.2 秒`
- 转盘结束后，立即显示最终保留的口味标签

示意流程：

```text
原味 + 草莓 + 抹茶
        ↓
统计权重：草莓 1，抹茶 1
        ↓
合并位置出现小转盘（0.2 秒）
        ↓
随机保留 1 种口味
        ↓
生成对应口味的新高一级食材
```

### 5. 权重示例

| 本次参与合并的口味         | 权重统计      | 结果            |
| ----------------- | --------- | ------------- |
| 原味 + 草莓 + 抹茶      | 草莓 1，抹茶 1 | 草莓/抹茶，各 50%   |
| 草莓 + 草莓 + 抹茶      | 草莓 2，抹茶 1 | 草莓 2/3，抹茶 1/3 |
| 原味 + 草莓 + 草莓 + 抹茶 | 草莓 2，抹茶 1 | 草莓 2/3，抹茶 1/3 |

补充说明：

- 原味不计入权重
- 只统计本次参与合并的食材，不跨连锁追溯
- 每次合并最多只保留 1 种口味标签

## 三、补位规则

### 1. 合并后的空缺处理

- 一组食材完成合并后，仅保留落点位置的新高等级食材。
- 其余参与合并的位置变为空格。
- 空格会触发棋盘内现有食材向上补位。

### 2. 上浮补位

- 本项目使用“上浮补位”，不是传统下落补位。
- 空位优先由其下方最近的食材向上移动填补。
- 若下方没有可上浮食材，则在最底部生成新食材并向上补位。

示意图：

```text
补位前                补位后

[ A ]                [ A ]
[   ]   <- 空位       [ C ]
[ C ]       上浮  →   [ D ]
[ D ]                [ 新 ]
```

### 3. 新食材生成规则

- 触发时机：某列最底部仍存在空位时
- 可生成等级：1 到 4 级
- 生成概率：按照关卡配置
- 安全规则：新食材不能直接生成立即三连

### 4. 死局处理

- 若补位完成后，棋盘不存在任何可交换并达成合并的组合，则触发自动洗牌。
- 洗牌后的棋盘必须同时满足：
  - 不直接触发合并
  - 至少存在 1 组可通过交换形成合并的组合

***

## 四、完整结算流程

```text
玩家交换
  ↓
检测是否形成三连
  ↓
合并并生成高一级食材
  ↓
若有多口味冲突，则按权重转盘决定保留口味
  ↓
清除空位并执行上浮补位
  ↓
检测是否产生新的三连
  ↓
有则继续连锁，无则本轮结束
```
