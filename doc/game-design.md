## 🔄 消除与合成规则

### 三连消除规则

**横向三连消除**:
```
消除前:                    消除后:
┌────┬────┬────┐          ┌────┬────┬────┐
│ 🌾 │ 🌾 │ 🌾 │    →     │    │ 🥡 │    │  ← 中间生成进阶食材
└────┴────┴────┘          └────┴────┴────┘
   第N行, 第M-1, M, M+1列
```

**竖向三连消除**:
```
消除前:                    消除后:
┌────┐                    ┌────┐
│ 🌾 │                    │    │
├────┤                    ├────┤
│ 🌾 │    →               │ 🥡 │  ← 中间生成进阶食材
├────┤                    ├────┤
│ 🌾 │                    │    │
└────┘                    └────┘
   第M列, 第N-1, N, N+1行
```

### 多连消除规则

**每个独立匹配组都会生成1个高一级食材；如果同一轮出现多个匹配组，它们会同时合成**

这里的“独立匹配组”指的是：
- 同一次交换后，盘面上彼此分开的有效三连/四连/五连区域
- 每个匹配组单独选择自己的合成位置，并各自产出1个进阶食材
- 同一匹配组内即使命中了4格、5格或十字/T字结构，也仍然只产出1个进阶食材

双合成示例：
```
交换前:
第2行: [ baking ][ wheat ][ flour ][ wheat ][ dough ]
第3行: [ toast  ][ flour ][ wheat ][ flour ][ baking ]

交换后:
第2行: [ baking ][ wheat ][ wheat ][ wheat ][ dough ]   → 3个1级，同时合成1个2级
第3行: [ toast  ][ flour ][ flour ][ flour ][ baking ]  → 3个2级，同时合成1个3级
```

```
三连消除:
┌────┬────┬────┐     ┌────┬────┬────┐
│ 🌾 │ 🌾 │ 🌾 │  →  │    │ 🥡 │    │  ← 中间生成1个进阶食材
└────┴────┴────┘     └────┴────┴────┘

四连消除:
┌────┬────┬────┬────┐     ┌────┬────┬────┬────┐
│ 🌾 │ 🌾 │ 🌾 │ 🌾 │  →  │    │    │ 🥡 │    │  ← 交换位置生成1个进阶食材
└────┴────┴────┴────┘     └────┴────┴────┴────┘

五连消除:
┌────┬────┬────┬────┬────┐     ┌────┬────┬────┬────┬────┐
│ 🌾 │ 🌾 │ 🌾 │ 🌾 │ 🌾 │  →  │    │    │ 🥡 │    │    │  ← 交换位置生成1个进阶食材
└────┴────┴────┴────┴────┘     └────┴────┴────┴────┴────┘
```

### 合成规则

| 消除类型 | 生成结果 | 位置 |
|----------|----------|------|
| 三连 | 1个进阶食材 | 交换触发位置 |
| 四连 | 1个进阶食材 | 交换触发位置 |
| 五连+ | 1个进阶食材 | 交换触发位置 |

补充说明：
- 如果一次交换只形成1个匹配组，就只生成1个进阶食材
- 如果一次交换形成多个独立匹配组，就同时生成多个进阶食材
- 合成位置优先取交换的两个格子；如果交换格子都不在该匹配组内，则取该匹配组的中心位置


### 顾客队列

```
┌──────────────────────────────────────────────┐
│      👤1         👤2         👤3             │
│    [🍞×2]      [🍵×1]      [🍓×3]            │
│      🐰          🦊          🐻               │
│    [耐心条]    [耐心条]    [耐心条]          │
└──────────────────────────────────────────────┘

```

### 礼盒自动交付机制

```typescript
// 礼盒合成后自动交付流程
function autoDeliverGiftBox(giftBox: GiftBox, customers: Customer[]): void {
  // 1. 查找匹配顾客
  const matchingCustomer = customers.find(c => 
    c.demand.type === giftBox.flavor && c.demand.count > 0
  );
  
  if (matchingCustomer) {
    // 2. 播放飞行动画
    animateGiftDelivery(giftBox, matchingCustomer);
    
    // 3. 减少需求数量
    matchingCustomer.demand.count--;
    
    // 4. 需求完成处理
    if (matchingCustomer.demand.count === 0) {
      // 发放奖励
      addCoins(matchingCustomer.reward);
      // 移除顾客
      removeCustomer(matchingCustomer);
      // 立即补上下一位顾客
      addNextQueuedCustomer();
    }
  } else {
    // 5. 无匹配顾客，保留在合成区
    giftBox.status = 'waiting';
    // 礼盒占1格，但不影响其他食材合成
  }
}
```

### 回合制耐心机制

- **单位**: 步，不是秒
- **扣减时机**: 每次有效交换后，当前在场顾客全部 `-1`
- **道具**: 不消耗顾客耐心
- **补位顾客**: 新上场顾客拥有1次免扣
- **耐心耗尽**: 顾客离开，但不扣金币、不扣分
- **关卡失败**: 无步可走且仍有未结算顾客
- **关卡结束**: 所有顾客都已被服务或已离场

### 失败条件

```
游戏失败触发:
└── 格子填满 且 无可用移动
    └── 提示使用洗牌道具
        ├── 有洗牌道具且已观看广告 → 洗牌继续
```

### 胜利条件

```
关卡结束:
└── 所有顾客都已结算
    ├── 服务成功的顾客越多，收益越高
    ├── 错过的顾客越多，收益越低
    └── 若无步可走且仍有顾客未结算，则立即失败
```
