# 吐司面包店 - 核心系统规则

## 核心玩法与对局胜负规则

### 胜利和失败条件

每局有根据配置文件的固定数量的顾客
开局时总共有3星
错过一个顾客减一星
当没有星星时，直接结束游戏，判定失败
当店内没有顾客时进行结算，判定成功，剩余的星星为其评价，即1星到3分

#### 耐心值计数规则

- **独立计数**：每个顾客独立耐心值
- **耐心值规则**：只有当玩家进行的是有效交换，并成功触发了合并结算后，前台顾客才会扣减耐心值；无效交换不扣耐心
- **加耐心值机制** 当用一次合成的同等级食材数量超过3个，场上每个用户的耐心值都 +1。

#### 2.3 耐心值阈值与状态触发

| 耐心值区间        | 顾客状态 | 对局影响              |
| ------------ | ---- | ----------------- |
| 耐心值 > 5      | 正常等待 | 无任何负面效果           |
| 3 < 耐心值 <= 5 | 轻微急躁 | 需求变黄              |
| 0 < 耐心值 <=3  | 暴躁状态 | 需求变红+敲桌子动画        |
| 耐心值 <=0      | 顾客离开 | 减一星，并检查是否触发对局失败逻辑 |

## 体力系统规则

严格遵循需求的基础设定，同时补充平衡限制，避免出现无限体力的设计漏洞，贴合休闲碎片化的游玩定位。

### 3.1 基础数值设定

- 初始体力上限：5点（新用户首次登录时10点，也就是溢出5点），全玩家初始一致
- 无论选择哪个已解锁区域，开启一局游戏固定消耗1点体力
- 对局开启后，无论最终胜利/失败，已消耗的体力均不返还
- 体力最小值：0点，体力＜=0 无法开始新对局
- 看广告可以补充一体力，一天最多通过这种模式补充1次

### 3.3 体力恢复规则

| 恢复途径   | 详细规则                                   |
| ------ | -------------------------------------- |
| 自然恢复   | 每2小时自动恢复1点体力，体力达到当前上限后停止恢复             |
| 广告额外恢复 | 观看广告单次获得1点体力，广告获取的体力可临时突破上限溢出，溢出部分优先消耗 |

## 金币奖励体系

### 金币核心产出途径

###  口味金币对应表

#### 订单基础金币（核心产出）

单局内每完成1个顾客订单，立即发放对应口味的基础金币（每个顾客了只能有一个口味或原味的一个礼盒需求），金额=该口味基础售价，参考[吐司面包店区域与口味解锁策划方案](./吐司面包店区域与口味解锁策划方案.md)，即时到账，不受最终星级影响。高等级区域的专属口味售价更高，基础收益更丰厚，激励玩家解锁新区域。

### 金币核心消耗路径

- 新区域解锁：核心消耗，解锁成本随区域推进指数递增
